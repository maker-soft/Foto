# Настройка безопасности и статистики

## 1. Обновление Supabase

1. Откройте Supabase → SQL Editor → New query.
2. Скопируйте весь файл `supabase/security_analytics_upgrade.sql`.
3. Нажмите Run.
4. Скрипт должен выполняться после создания пользователя `lena.foto@mail.ru` в Authentication → Users.
5. Если пользователь был создан позже, повторно выполните блок `insert into private.site_admins ...` из начала SQL-файла.

Скрипт добавляет:
- доступ к админке только для UUID разрешённого пользователя;
- закрытые таблицы статистики и событий безопасности;
- серверные функции сохранения контента и получения статистики;
- ограничение размера и типов загружаемых изображений;
- RLS-политики для контента и Storage.

## 2. Защита от перебора пароля

В коде включена локальная задержка после пяти ошибок. Supabase также применяет серверные ограничения Auth.

Для максимальной защиты включите Cloudflare Turnstile:

1. Создайте сайт в Cloudflare Turnstile и добавьте домен GitHub Pages.
2. В Supabase откройте Authentication → Bot and Abuse Protection.
3. Включите CAPTCHA, выберите Cloudflare Turnstile и вставьте Secret Key.
4. В существующий `assets/js/config.js` добавьте:

```js
CAPTCHA_SITE_KEY: "ВАШ_PUBLIC_SITE_KEY",
REQUIRE_CAPTCHA: true,
MAX_UPLOAD_BYTES: 10485760
```

Не заменяйте рабочие `SUPABASE_URL` и `SUPABASE_ANON_KEY`. Никогда не размещайте `service_role` или Secret Key на GitHub.

## 3. Статистика

После обновления SQL в админке появится раздел «Статистика»:
- просмотры и сеансы;
- статистика за 7, 30 и 90 дней;
- популярные страницы;
- устройства и источники переходов;
- число неудачных и заблокированных попыток входа.

Статистика не сохраняет IP-адреса, пароли, ФИО и содержимое форм.

## 4. Рекомендуемые настройки

- Установите администратору уникальный пароль длиной не менее 14 символов.
- Не создавайте дополнительных пользователей Supabase без необходимости.
- Включите CAPTCHA до публикации адреса админки.
- Периодически проверяйте раздел «Статистика → Безопасность входа».
- Храните резервную копию таблицы `site_content`.
